Show, don't tell

One command. Full picture.

Drop into any project and run it. viscacha walks the directory, finds every version file, checks what you have installed, and shows you where things drift.

~/work/monorepo
$ viscacha
Tool Required Installed Source Status
─────────────────────────────────────────────────────────
node 20 20.11.0 .nvmrc ✓
python 3.11 3.11.4 .python-version ✓
rust 1.76.0 1.80.0 rust-toolchain.toml ✗
go 1.22.0 1.22.0 go.mod ✓
node >=18 20.11.0 package.json ✓
npm >=9 10.2.4 package.json ✓
5 passing, 1 failing

Something drifted? Ask for fixes.

~/work/monorepo
$ viscacha --fix
Tool Required Installed Source Status
───────────────────────────────────────────────────────
rust 1.76.0 1.80.0 rust-toolchain.toml ✗
0 passing, 1 failing
Suggested fixes:
rust: rustup toolchain install 1.76.0 && rustup override set 1.76.0

Install

Three ways to get the binary on your PATH.

$ cargo install viscacha

Requires a recent stable Rust toolchain.

$ git clone https://github.com/iamkorun/viscacha.git
$ cd viscacha
$ cargo install --path .

Builds from main. Takes about as long as a fresh coffee.

What it does

Four things. Nothing else.

01 · read

Eight version file formats in one scan.

Dig through .nvmrc, .node-version, .tool-versions, rust-toolchain.toml, rust-toolchain, .python-version, go.mod, and package.json. If your project uses it, viscacha reads it.

No config file. No plugin registry. It just walks the directory.
$ ls .n* rust-* *.mod
.node-version rust-toolchain.toml
.nvmrc go.mod
.python-version
$ viscacha --verbose
found 5 source files
checking 6 constraints
$ viscacha --fix
Suggested fixes:
node: nvm install 20
rust: rustup toolchain install 1.76.0
python: pyenv install 3.11
02 · fix

The exact commands to unstick yourself.

Pass --fix and viscacha prints the real shell commands for nvm, rustup, pyenv, or asdf — whichever fits the mismatch. Copy, paste, done.

No more googling "how to install rust 1.76.0" at 9pm on a Friday.
03 · gate

Drop it into CI. Exit codes do the rest.

Quiet mode speaks only in exit codes. 0 when everything matches, 1 when something drifts. Your GitHub Actions step either passes or it doesn't.

Catch "works on my machine" before the build does.
.github/workflows/ci.yml
jobs:
check-toolchain:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v4
- run: cargo install viscacha
- run: viscacha --quiet

04 · fast

Written in Rust. Done in milliseconds.

Single static binary, no Node runtime, no Python interpreter, no JVM spin-up. viscacha finishes faster than ls on a cold cache.

Add it to a shell hook. You won't feel it.
